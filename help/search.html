The script search box allows you to find definitions, both currently active and inactive, based upon various characteristics such as observable name or dependencies. Typing a word in the box will match all observable names that are similar to that, a drop down box then contains suggestions.

Hashtags:

If a definition has a comment with it that contains hashtags, then search for #thathashtag will show only those definitions matching that tag.

Restricting results:

The results can be filtered by starting the search with one of the following tokens followed by a space and your query.

  • "active: " - Show only those definitions and agents that are currently active
  • "inactive: " - Show only those that are not active
  • "agents: " - Only display agents
  • "handles: " - Select only assignments or definitions that are just a single literal value
  • "oracles: " - Select definitions that are more complex expressions
  • "defs: " - Only display definitions and no agents
  • "shared: " - Only display definitions in the cloud that I do not have locally already

These filters can be combined to, for example, select only active agents using "active:agents: ".

Dependency Searches:

Statements can be search based upon what they are dependant on or what is determined by something. Search tokens are added before the observable name to match, with no space between the : and the word to match.

  • "depends:" - Find all results that directly depend upon some other observable
  • "determines:" - Find all results that directly and indirectly determine some other observable

Sorting:

The results can be sorted by several properties such as time or name. This is specified by including one of the following search tokens somewhere in the search.

  • "sort:youngest" - Put the most recent definition first
  • "sort:oldest" - Put the oldest definition first
  • "sort:name" - Sort alphabetically by observable name
